¿Cómo utilizarlo para envio de información de manera segura?

Los pasos serían los siguientes:

  1. Encriptación
  2. Encapsulación
  3. Adición de encabezado
  4. Transmisión
  5. Desencapsulación
  6. Desencriptación
  7. Entrega

El tunneling implica seis procesos clave: encriptación, encapsulación, adición de encabezado, transmisión, desencapsulación y desencriptación. Estos procesos ocurren tanto para el tráfico saliente como para el entrante.

  1. Encriptación : Los bits de datos conocidos como "paquetes" se encriptan para asegurar la seguridad y privacidad durante la transmisión.

  2. Encapsulación : Los paquetes de datos encriptados se encapsulan dentro de un protocolo de red pública, un conjunto de reglas que gobiernan la comunicación y transmisión de datos en una red, y se convierten en el "payload" —los datos que se transmiten de forma segura.

  3. Adición de encabezado : Un encabezado especifica un extremo del túnel seguro como punto de origen y el otro como destino. Durante la encapsulación, un paquete de datos encriptados se envuelve en un protocolo de envoltura que añade un encabezado. En términos simples, este paso significa que el encabezado indica a los datos a dónde ir.

  4. Transmisión : El paquete encapsulado (disfrazado) atraviesa la red pública sin ser notado.

  5. Desencapsulación : Al llegar a la red local, los datos pasan por un proceso de desencapsulación. Este proceso implica la eliminación del encabezado añadido durante la fase de encapsulación. La desencapsulación revela el paquete de datos original que estaba encriptado, preparándolo para el siguiente paso del proceso.

  6. Desencriptación : Una vez que el paquete de datos ha sido desencapsulado, el siguiente paso es la desencriptación. En esta etapa, el paquete de datos encriptado se transforma de nuevo a su formato original. Este proceso utiliza una clave de desencriptación, que es conocida solo por el remitente y el receptor, asegurando que solo las partes autorizadas puedan acceder a la información contenida en el paquete.

  7. Entrega : Después de la desencriptación, los datos están ahora en un estado legible y seguro. El paquete es entonces entregado a la dirección IP de destino prevista. Este destino es típicamente un dispositivo o servidor dentro de la red privada del receptor, completando así el viaje seguro del paquete de datos a través de la red pública.

¿Qué protocolos se usan para el tunneling?

Los protocolos de tunneling más comunes son:

  • Point-to-Point Tunneling Protocol (PPTP)
  • Layer 2 Tunneling Protocol (L2TP)
  • Internet Protocol Security (IPSec)
  • Secure Shell (SSH)
  • OpenVPN
  • Secure Socket Tunneling Protocol (SSTP)

¿Qué es IPsec?

IPSec es un conjunto de reglas o protocolos de comunicación para configurar conexiones seguras a través de una red. El protocolo de Internet (IP) es el estándar común que determina cómo viajan los datos por Internet. IPSec agrega cifrado y autenticación para hacer que el protocolo sea más seguro. Por ejemplo, IPSec codifica los datos en el origen y los descodifica en su destino. También autentica el origen de los datos.
